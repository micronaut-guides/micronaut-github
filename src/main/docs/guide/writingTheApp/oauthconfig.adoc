<1> The provider identifier should match the last part of the url you entered as a redirect url `/oauth/callback/github`
<2> Client Secret. See previous screenshot.
<3> Client ID. See previous screenshot.
<4> Specify the scopes you want to request. https://developer.github.com/apps/building-oauth-apps/understanding-scopes-for-oauth-apps/[Github Scopes] let you sepcify exactly what type of access you need.
<5> Map manually the Github's https://tools.ietf.org/html/rfc6749#section-3.1[Authorization endpoint].
<6> Map manually the Github's https://tools.ietf.org/html/rfc6749#section-3.2[Token endpoint].
<7> Once validated, we are going to save the ID Token in a Cookie. To read in subsequent requests, enable https://micronaut-projects.github.io/micronaut-security/latest/guide/index.html#cookieToken[Cookie Token Reader].
<9> Enable https://micronaut-projects.github.io/micronaut-security/latest/guide/index.html#logout[Logout Controller]
<9> Accept GET request to the `/logout` endpoint.

The previous configuration uses several placeholders. You will need to setup `OAUTH_CLIENT_ID`, `OAUTH_CLIENT_SECRET` environment variables.

[soruce, bash]
----
export OAUTH_CLIENT_ID=XXXXXXXXXX
export OAUTH_CLIENT_SECRET=YYYYYYYYYY
----
